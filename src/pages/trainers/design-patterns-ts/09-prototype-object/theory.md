---
title: "Паттерн Prototype (Прототип), практика: класс-прототип объекта"
---

Как мы уже увидели, паттерн Prototype реализуется посредством нативной фичи TS — прототипного наследования. Классы в TS — это синтаксический сахар, заменяющий ту же механику с функциями и их прототипами.

Представьте, что есть задача: написать класс-прототип `User` пользователя с базовой логикой.

Класс полжен принимать в конструкторе объект с данными пользователя — `data`. Данные могут включать следующие поля:

- `id` — уникальный идентификатор
- `firstName` — имя пользователя
- `lastName` — фамилия пользователя
- `email` — email-адрес
- `role` — роль в системе (admin/user/etc)

В конструкторе `User` объект с данными должен «распаковываться», и полям класса должны явно присваиваться значения:

```js
/**
 * Создает экземпляр пользователя
 * @param {object} data — объект с данными пользователя
 * @param {string} data.id — уникальный идентификатор
 * @param {string} data.firstName — имя пользователя
 * @param {string} data.lastName — фамилия пользователя
 * @param {string} data.email — email-адрес
 * @param {string} data.role — роль в системе (admin/user/etc)
 */
constructor(data) {
  const { id } = data;
  this.id = id;
  // ...
}
```

Также класс должен содержать следующие поля и методы:

- геттер `fullName` должен возвращать объединённую строку в формате `Имя Фамилия` пользователя через пробел:

```js
/**
 * Полное имя пользователя (геттер)
 * @returns {string} Объединенное имя и фамилия
 * @readonly
 */
get fullName() {}
```

- метод `isAdmin()` должен возвращать булево поле `true` в случае, если `role === 'admin'` и `false` в остальных случаях:

```js
/**
 * Проверка прав администратора
 * @returns {boolean} true если пользователь имеет роль admin
 */
isAdmin() {}
```

Реализуйте в модуле `user.js` класс `User` согласно этим условиям.
