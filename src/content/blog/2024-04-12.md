**#Пульс*веб*платформы 12.04.2024**

**Новости**

- [вышел Jpegli](https://opensource.googleblog.com/2024/04/introducing-jpegli-new-jpeg-coding-library.html?ck_subscriber_id=2011909133) — новая либа для кодирования jpeg, обещают на 35% лучшее <s>зашакаливание</s> сжатие, казалось бы jpeg сто лет в обед, но до сих пор случаются прорывы
- [PartyKit](https://www.partykit.io/), библиотека для создания мультиплеер-приложений, [куплена Cloudflare](https://blog.cloudflare.com/cloudflare-acquires-partykit): обещают холить и лелеять (но это в пресс-релизах они все так обещают)
- [V8 Sandbox](https://v8.dev/blog/sandbox) — более не эксперимент: уже в Chrome 123 память для V8 будет отделяться в отдельную «резервацию», чтобы предотвратить несанкционированный доступ к ней
- [вышел](https://web.dev/blog/introducing-learn-javascript) новый [учебник по JS](https://web.dev/learn/javascript/control-flow) от гугловцев: как обычно, считаю, что только читать материал — мало, квизы в конце статей — тоже мало, надо отрабатывать на практике, вот буквально набивать руку, печатая код
- [вышел ESLint v9.0.0](https://eslint.org/blog/2024/04/eslint-v9.0.0-released/), в нём много нововведений и breaking changes: не поддерживается нода < v18.18.0, v19; eslint.config.js — теперь дефолт, .eslintrc —депрекейт; удалены форматтеры, кроме stylish, html, json и json-with-meta; обновлен eslint:recommended; добавлены и изменены многие правила
- вышел [ESLint Config Inspector](https://eslint.org/blog/2024/04/eslint-config-inspector/) — инструмент для визуального изучения вашего конфига, к чему он применяется, почему, зачем, за что (работает только с eslint.config.js) — это приятная пилюля для тех, кто страдает, что надо снова всё перелопачивать

**Проекты**

- [ts-pattern](https://github.com/gvergnaud/ts-pattern) — либа реализующая pattern matching подход в TS: если у вас есть развесистые условия, то возможно стоит присмотреться к либе
- [vinejs](https://vinejs.dev/) — если вдруг вы использовали Zod в Node для валидации полей формы с клиента, то вас стоит взглянуть на спец либу для валидации именно HTTP request body
- [dioma](https://github.com/zheksoon/dioma) — либа для реализации dependency injection в JS и TS
- [cally](https://github.com/WickyNilliams/cally) — веб-компонент календаря: фичастый, маленький, доступный, твой
- [previewjs](https://previewjs.com/) — таб с превью отрендеренного компонента из текущего файла прямо в редакторе для React, Vue, Svelte, Solid

**Статьи и демки**

**JS**

- очередное напоминание о существовании Shadow DOM без разглагольствований: [бери и копируй базовый веб-компонент кнопки](https://southleft.com/insights/development/adopted-stylesheets/), который сразу заработает
- [шоукейс библиотеки Effect](https://dev.to/almaju/someone-finally-fixed-javascript-426i), которая берёт на себя задачу помочь декларативностью и наглядностью побороть неоптимальную конкуренцию в асинхронных приложениях
- [оптимизация бандла](https://dev.to/appqui/how-i-optimized-carousel-for-editorjs-2x-in-size-4efa), в общем-то, неважно какого приложения: основная мысль — бандл — не бинарник, это высокоуровневый JS, который можно растрепать и посмотреть, что там есть лишнего (webpack-bundle-analyzer в помощь)
- [yarn для тех](https://habr.com/ru/companies/dododev/articles/798519/), кому нужно не только запускать команды из package.json, а: хранить кэш всех пакетов проекта в репозитории; использовать workspaces в монорепе; интерактивно обновлять зависимости проекта; гибко управлять версиями пакетов; использовать «протоколы» для подключения к проекту внешних файлов и папок; патчить пакеты и другое

**CSS**

- [крутые текстовые эффекты](https://texteffects.dev/) (ведь всю эту мощь в CSS же принесли для стилей, а не программирования, так?)
- что будет, если соединить :has(:hover) и сетку на 100500 элементов — [позиционирование курсора без использования JS](https://dev.to/warkentien2/curious-geckos-the-most-precise-css-only-position-aware-mini-game-16i2)
- а что будет, если соединить :has(input:focus) input:invalid и пачку инпутов с maxlength="1" и pattern="[tT]" — [wordle без использования JS](https://scottjehl.com/posts/wordleish/)
- [«складывающиеся» элементы](https://www.joshwcomeau.com/react/folding-the-dom/) на transform-ах: особо приятен эффект затенения при более остром угле складывания
- использование CSS как инструмента [для причинения умышленного визуального регресса](https://heydonworks.com/article/testing-html-with-modern-css/) для антипаттернов в HTML (например, для выявления забытых пустых ссылок a:not([href]), a[href=""], a[href$="#"], a[href^="javascript"])

**HTML**

- ещё один [гайд по <img>, srcset, sizes и <picture>](https://danburzo.ro/responsive-images-html/) с фокусом на последовательное понимание, а зачем оно вообще нужно

**Платформа**

- если что WebP уже не самый топ-формат и повсеместно [распространён более лучший AVIF](https://blog.sentry.io/low-effort-image-optimization-tips/), правда руками настраивать всю машинерию _имаджинерии_ сложно, поэтому можно воспользоваться инструментами для пребилда картинок в нужном формате или доставкой из CDN
