### Новости

- появился новый линтер [oxlint](https://oxc-project.github.io/blog/2023-12-12-announcing-oxlint.html):
  - написан на Rust
  - можно не настраивать (convention over configuration)
  - плагинов пока нет, но обещают, что можно будет писать их без использования JS или Rust
  - быстр, но в том числе потому что проверяемых правил мало
- вышел [SvelteKit 2](https://svelte.dev/blog/sveltekit-2): [Vite 5](https://vitejs.dev/blog/announcing-vite5) под капотом, подготовка к будущему Svelte 5, [shallow routing](https://kit.svelte.dev/docs/shallow-routing); tldr: Svetle продолжает прорастать в экосистему
- [Vue 2 станет deprecated](https://blog.vuejs.org/posts/vue-2-eol) 31 декабря 2023
- вышел [Firefox 121](https://www.mozilla.org/en-US/firefox/121.0/releasenotes/): главное поддержан `:has()`!
- вышла [библиотека date-fns v3](https://blog.date-fns.org/v3-is-out/): теперь 100% типизирована, уменьшен размер бандла, функции экспортируются через named exports и кое-что ещё
- обновился [Tailwind CSS v3.4](https://tailwindcss.com/blog/tailwindcss-v3-4): динамические viewport units, поддержка `:has()`, `text-wrap: balance`, субгриды и многое другое

### Проекты

- [waku](https://github.com/dai-shi/waku) — минималистичный React-фреймворк с поддержкой ServerComponents: включает клиент, сервер и роутер
- [constate](https://github.com/diegohaz/constate): библиотека для решения проблемы плодящихся контекстов в Реакте
- [simple-web-server](https://github.com/terreng/simple-web-server): простая декстопная аппка для настройки и запуска дев-сервера

### Статьи и демки

#### JS

- [список](https://gist.github.com/guest271314/bd292fc33e1b30dede0643a283fadc6a) всех известных (и неизвестных) JS рантаймов и движков на все случаи жизни (даже чтобы выполнять JS в с/с++)
- если промис ушёл и не вернулся, он попал в чёрную дыру: чтобы это исправить нужно [отказаться от асинхронных функций в угоду генераторов](https://frontside.com/blog/2023-12-11-await-event-horizon/)
- [профилируя приложение для локализации](https://calendar.perfplanet.com/2023/benchmarking-profiling-and-optimizing-javascript-libraries/) автор пришёл к тому, что `node --prof` и инструмент [speedscope](https://www.npmjs.com/package/speedscope) — хорошая отправная точка для анализа перфоманса
- члены tc39 устали отвечать на одни и те же вопросы в ишью, и [создали faq](https://github.com/tc39/faq/), в котором можно узнать, почему [JSX не появится в ES](https://github.com/tc39/faq/#can-we-change-json), почему [JSON никогда не поменяется](https://github.com/tc39/faq/#can-we-change-json), а также [почему числа в JS такие странные](https://github.com/tc39/faq/)
- в [конкурсе](https://calendar.perfplanet.com/2023/fastest-way-passing-state-javascript-revisited/) на самый быстрый способ проброса состояния с сервера на клиент победил `<script type="mime/invalid" id="myState">{"foo":"bar"}</script> window.__STATE__ = JSON.parse(window.myState.innerHTML)`
- [темы для холиваров](https://www.bekk.christmas/post/2023/13/react-fights-you-can-have-with-your-team) в вашей команде, если вы работатете с Реактом: как писать экспорты, как называть файлы и пропсы, нужны ли стрелочные функции, а также тернарники vs early return
- [TS-фичи](https://medium.com/@hamsterhomka/typescript-features-that-will-significantly-upgrade-your-skills-e40a3424fa50) «следующего» уровня: satisfies, infer, exhaustive type checking
- эволюция [синхронного рекурсивного обхода папок в ноде](https://gist.github.com/kethinov/6658166) достигла вершины — `await fs.readdir(dir, { recursive: true })`
- [всё про Unicode для разработчиков](https://dmitripavlutin.com/what-every-javascript-developer-should-know-about-unicode/): символы, их объединение, эскейп в JS, сравнение строк и регекспы

- [лок скролла из глубины DOM-а](https://robbowen.digital/wrote-about/locking-scroll-with-has/) с помощью `body:has(.lock-scroll) { overflow: hidden} <dialog class="lock-scroll">`
- [четыре способа](https://bejamas.io/blog/css-animated-gradient-border/) создать анимированные градиентные рамки
