### Новости

- [вышел Chrome 135](https://developer.chrome.com/release-notes/135):
  - появились атрибуты `command` и `commandfor` для кнопок, чтобы декларативно описывать взаимодействие с поповерами и диалогами
  - добавили псевдоэлементы `::scroll-button()` и `::scroll-marker` для «нативных» каруселей
  - CSS-функция `shape()` позволяет описывать в `clip-path` параметрические SVG-like-формы
  - добавлена возможность стилизации псевдоэлемента внутри псевдоэлемента `::before::marker`
  - поддержали [Observable API](https://wicg.github.io/observable/) и `fetchLater()` для отложенного запуска `fetch`
- [вышел Safari 18.4](https://webkit.org/blog/16574/webkit-features-in-safari-18-4/):
  - внедрили новый декларативный формат Web Push
  - также поддержали CSS-функцию `shape()`
  - обновили `<input type="color" />`, теперь поддерживает атрибуты `alpha` и `colorspace`
  - поддержали [Cookie Store API](https://developer.mozilla.org/en-US/docs/Web/API/Cookie_Store_API)
- [вышел Firefox 137](https://www.mozilla.org/en-US/firefox/137.0/releasenotes/): из интересного только выкатили группировку табов, а в ночных сборках [появляется поддержка PWA](https://www.omgubuntu.co.uk/2025/03/firefox-nightly-supports-web-apps-taskbar-tabs)
- [выпустили React 19.1.0](https://github.com/facebook/react/releases/tag/v19.1.0): появилась фича Owner Stack для более детального отслеживания в дев-режиме иерархии рендера компонентов, а также подфиксили баги в React и React DOM
- вышел [Express 5.1.0](https://expressjs.com/2025/03/31/v5-1-latest-release.html), эта версия становится LTS
- [поддержка Node.js 18](https://nodejs.org/en/blog/release/v18.20.8) заканчивается 30 апреля 2025, время обновить ваши `.nvmrc`
- [выпущен Material UI v7](https://mui.com/blog/material-ui-v7-is-here/):
  - улучшилась поддержка ESM
  - устаканилось API `slots` и `slotProps`
  - впилили поддержку `@layer` для упрощения интеграции с другими либами
- [вышел отчёт State of Vue.js 2025](https://www.monterail.com/stateofvue):
  - хотят по-прежнему дожать Vapor Mode с обратной совместимостью
  - по популярности Vue всё ещё отстаёт от React, но количество сайтов на Vue явно больше Angular
  - треть проектов всё ещё на Vue 2, хоть он больше и не поддерживается (миграция вызывает сложности)
  - Pinia — доминирующий стейт-менеджер
  - комьюнити не хватает библиотек компонентов (как MUI или Radix) под Vue
  - Nuxt v4 уже неподалёку
- со стороны WebKit [появилось предложение](https://webkit.org/blog/16587/item-flow-part-1-a-new-unified-concept-for-layout/) объединить пересекающиеся части flexbox и grid в новом семействе свойств `item-direction`, `item-wrap`, `item-pack`, `item-slack` (чтобы наконец перестать вспоминать каждый раз, что эти `align-smthng` значат)

### Статьи и демки

#### JS

- [кольцевые зависимости между модулями в JS](https://www.bryanbraun.com/2025/03/29/breaking-down-circular-dependencies-javascript/) не отлавливаются на уровне языка/среды, но они могут потенциально приводить к ошибкам типа `ReferenceError: Cannot access 'a' before initialization`, так как к содержимому из одного модуля может обращаться другой модуль до его инициализации и полного выполнения; беспорядочные кросс-импорты в «шареной» зоне проекта нуждаются в особом внимании, для этого есть сторонние либы для отслеживания корректности импортов [madge](https://github.com/pahen/madge) и [eslint-plugin-import](https://www.npmjs.com/package/eslint-plugin-import)
- [мемоизация в React](https://cekrem.github.io/posts/react-memo-when-it-helps-when-it-hurts/) изначально спроектирована довольно хрупко: простое заворачивание пропсов в `useMemo`/`useCallback` не повлияет на ререндеры компонента, нужно ещё замемоизировать сам компонент в `React.memo`, а также следить, что внутри нет спреда пропсов `<Child {...props} />`, нет немемоизированного прокидывания `{ children }` (они пересоздаются каждый раз), нет дополнительных дочерних компонентов без мемоизации

#### CSS

- [аутлайн](https://adactio.com/notes/21812) по клавиатурному фокусу на ссылках никогда не повредит:

```css
a:focus-visible {
  outline-offset: 0.25em;
  outline-width: 0.25em;
  outline-color: currentColor;
}
```

- [про тени](https://frontendmasters.com/blog/expanding-css-shadow-effects/): три вида теней — `box-shadow`, `filter: drop-shadow()` и `text-shadow` —могут не только статически применяться, но и анимироваться, причём благодаря `@property` возможна анимация только одного из параметра тени, например, `box-shadow: inset 0 0 0 var(--l) var(--c)`, что делает тени мощным декоративным элементом
- прикольный, но хрупкий [голографический эффект на скролле](https://codepen.io/HejChristian/full/YPzLbYX), построенный на режимах наложения, `background-attachment: fixed` и линейных градиентах
- а пока создатели CSS размышляют не схлопнуть ли flex и grid в единое свойство, разработчикам как-то нужно жить и решать, что использовать в каждом конкретном юзкейсе: вот у одного есть мнение, что в решении, что взять, нужно следовать в порядке [grid/inline-grid → block → flex](https://alex.party/posts/2025-03-23-grid-first-flex-third/)
- об анимации движения по линии с помощью `offset-path` вы уже наверняка слышали, но тут [идею развили поглубже](https://yuanchuan.dev/css-animation-with-offset-path) и творят прям интересные экземпляры (правда с использованием [css-doodle](https://github.com/css-doodle/css-doodle)), так что есть чем вдохновиться
