### Новости

- [обновился редактор Cursor 1.0](https://www.cursor.com/changelog/1-0):
  - главное — появилась настройка в 1 клик MCP-серверов
  - также из интересного появилась бета-фича Memories: редактор может запоминать факты из диалогов и ссылаться на них в будущем
- [последние обновления ES от TC39](https://socket.dev/blog/tc39-advances-9-proposals):
  - `Array.fromAsync` перешёл на Stage 4 (доступен везде)
  - `Error.isError` перешёл на Stage 4 (доступен везде)
  - `using` перешёл на Stage 4 (Chrome only)
  -  [Seeded Pseudo Random Numbers](https://github.com/tc39/proposal-seeded-random) от Tab Atkins перешёл на Stage 2
- [обновился Astro 5.9](https://astro.build/blog/astro-590/):
  - появилась экспериментальная встроенная поддержка `Content-Security-Policy` посредством встраивания тега `<meta>` с нужными хэшами используемых файлов (также загружаемых динамически)
- недавно анонсировали [`@platformatic/php-node`](https://github.com/platformatic/php-node) — официальный модуль Node.js с возможностью процессить PHP внутри Node.js-рантайма; главный вопрос был «зочем?», и вот [выпустили статью](https://blog.platformatic.dev/seamlessly-blend-php-with-nodejs), где рассказали юзкейсы: миграция PHP-прил на Node.js, гибридные приложения (Node.js сервисы с кусочками PHP-процессинга)
- [React Router пообещали сделать стабильнее](https://remix.run/blog/rr-governance) через перевод процесса разработки на процесс [TC39 Process](https://tc39.es/process-document/) (проход по стейджам + публичные обсуждения)
- команда Edge завезла в будущий Chrome 139 [стилизацию gap-ов в гридах/флексбоксах](https://developer.chrome.com/blog/gap-decorations) посредством новых свойств `column-rule-*` и `row-rule-*`
- [релизнули первый стабильный Oxlint](https://voidzero.dev/posts/announcing-oxlint-1-stable) (линтер на Rust от Evan You сотоварищи):
  - за счёт мультипоточности быстрее ESLint в 50-100x
  - есть автомиграция flat-config-а ESLint в `.oxlintrc.json`
  - сразу включает > 500 правил из самого ESLint и популярных плагинов + собственные правила
- [DigitalOcean выпустили MCP-сервер](https://www.digitalocean.com/community/tutorials/control-apps-using-mcp-server), чтобы деплоить на платформу сразу из редактора без лишних конфигов
- [анонсировали бету Safari 26](https://webkit.org/blog/16993/news-from-wwdc25-web-technology-coming-this-fall-in-safari-26-beta/), да, Safari 19 не будет, будет сразу 26, следуя за числом года (что будет после 2099 года видимо уже неважно):
  - наконец-то будут SVG-favicon-ки
  - тег `<model>` для показа 3d-моделей в браузере (с фолбеком в обычную картинку)
  - в CSS поддержаны Anchor Positioning, Scroll-driven Animations, `contrast-color()`, `progress()`, `margin-trim`
  - из API поддержаны `URLPattern`, `scrollMargin` в `IntersectionObserver`, `File System WritableStream`,  [Pattern Modifiers](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Regular_expressions/Modifier) в `RegExp`

### Статьи, мнения, туториалы

#### JS/TS

- [с помощью `document.currentScript`](https://macarthur.me/posts/current-script/) можно пробросить через тег `<script>` внутрь скрипта data-атрибуты, чекать обычные атрибуты (типа, что скрипту обязательно проставлен `defer`) или же порядок следования скрипта в разметке (только в начале `body` или после определённого элемента)
- Tanner Linsley убеждает, что [URL как хранилище стейта недооценено](https://tanstack.com/blog/search-params-are-state) из-за проблем с парсингом, типизацией и отсутствия связи с роутингом напрямую и рекомендует использовать TanStack Router, чтобы порешать все эти проблемы и наслаждаться рантайм-валидацией роутов с безопасным сохранением стейта в URL
- в отличие от директив (типа `use strict`) магические комменты в начале файлов вроде `/** @aPragma */` или `//# aMagicComment`, влияющие на процесс интерпретации и транспиляции JS-файлов, [нестандартны и мало задокументированы](https://macwright.com/2025/04/29/directive-prologues-and-javascript-dark-matter)

#### CSS

- [напоминание](https://frontendmasters.com/blog/scroll-driven-letter-grid/), что у вариативных шрифтов можно анимировать `font-weight`
- [ещё одно напоминание](https://web-platform-dx.github.io/web-features-explorer/features/viewport-unit-variants/): вьюпорт-ориентированные динамические единицы измерения `sv*`, `lv*`, `dv*` с нами уже с 2022 года, то есть сейчас уже считаются широко распространёнными
- анимация до не фиксированного, а [динамического значения](https://www.joshwcomeau.com/animation/partial-keyframes/):

```css
@keyframes oscillate {
  from {
    transform: translateX(calc(var(--amount) * -1));
  }
  to {
    transform: translateX(var(--amount));
  }
}
```

- `1fr 1fr` vs `auto auto` vs `50% 50%`, [что выбрать, если нужно разделить на две равные части](https://frontendmasters.com/blog/1fr-1fr-vs-auto-auto-vs-50-50/): в итоге лучше будет воспользоваться гридовым `minmax(0, 1fr)` для возможности работы `overflow` при переполнении

#### HTML

- элемент `<output>` [задуман для отображения результата вычисления](https://html-css-tip-of-the-week.netlify.app/tip/output/), значение задаётся как для элемента формы (через `value`), но при этом находясь внутри `<form>` элемент не сабмитится, то есть служит чисто для визуализации
