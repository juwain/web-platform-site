**Новости**

- на волне новостей о React Compiler [анонсирован выпуск Million Lint 1.0-rc](https://million.dev/blog/lint-rc): в отличие от компайлера, который пребилдит реакт-код, Million Lint втыкает в код точечно замеры производительности и уже на их основе выдаёт рекомендации в редакторе кода и подсвечивает проблемы с производительностью
- на той же волне [анонсирован Next.js 15 RC](https://nextjs.org/blog/next-15-rc): поддерживает React 19 RC, React Compiler, произошёл отказ от кеширования по умолчанию в fetch, GET Route Handlers и роутере, постепенно выкатывается реактовская система частичного пререндера Partial Prerendering (PPR) и after() API для отложенного запуска второстепенных тасков после выполнения основного запроса
- и снова на той же самой волне [вышел SolidStart v1.0.0](https://github.com/solidjs/solid-start/releases/tag/v1.0.0), метафреймворк на основе Solid, встроенного роутера, а также Vite, Nitro и другого опенсорса; конечно, не перестаю удивляться современной индустрии, которую [люди из опенсорса натурально тащат вперёд](https://www.solidjs.com/blog/solid-start-the-shape-frameworks-to-come) на энтузиазме и за счёт синергии составляют конкуренцию корпорациям

**Проекты**

- [Responsive Viewer](https://responsiveviewer.org/) — если много верстаете или тестируете на разных разрешениях, это плагин для браузера, чтобы одновременно на одном экране открыть несколько «девайсов» с синхронизацией скролла и кликов
- [matcha.css](https://github.com/lowlighter/matcha) — CSS-либа для стилизации чистой семантической разметки + небольшой набор классов хелперов для стилизации лейаута, цветов и подсветки синтаксиса (внутри нет `!important`-ов, то есть всё переопределяется пользовательскими стилями)
- [open-props](https://open-props.style/) — коробка с CSS-переменными на все случаи жизни (цвета, размеры, шрифты, тени, скругления)
- [react-google-maps](https://visgl.github.io/react-google-maps/) — реакт-компоненты и хуки для Google Maps JavaScript API
- [tsx](https://tsx.is/) — запускальщик ts-файлов в Node (одной строкой и без конфигов, бонусом есть вотчер)
- [clack](https://github.com/natemoo-re/clack/tree/main/packages/prompts#readme) — либа для создания красивых и удобных cli-промптов

**Статьи и демки**

**JS**

- иногда для повышения читаемости можно [сделать код более «композируемым», но повторяющимся](https://kyleshevlin.com/prefer-multiple-compositions/) (на примере реакт-компонентов), лишним `if` или `switch` кашу не испортишь
- [Solid мимикрирует под реактовский JSX](https://www.stashpad.com/blog/react-developer-guide-to-solid-js), но под капотом компонент в React — это функция, которая вызывается каждый раз при рендере, а в Solid — один раз и возвращает DOM-элемент; вместо `useState` используется `createSignal`, который тоже похож внешне и возвращает массив с функцией доступа к состоянию и сеттер, но на деле не вызовет ререндера компонента, а только изменит контент в DOM-ноде; также в Solid есть `createEffect` (похож на `useEffect`), в который в простейшем случае не нужно передавать зависимости, что может привести к неочевидным багам
- (я из 2012 понимает всю боль) реактивность — это прежде всего [способ избежать связанности (coupling) кода](https://habr.com/ru/companies/ruvds/articles/737114/)

**CSS**

- ещё одно напоминание, что каскадные слои можно использовать для организации типов стилей, например, отдельно объявлять `@layer reset, theme, layout`, причём подключаться они будут в таком порядке независимо от порядка подключения в коде; также описан концепт, как через «ручки» (кастомные свойства) [управлять параметрами лейаута страницы](https://www.smashingmagazine.com/2024/05/modern-css-layouts-no-framework-needed/)
- [рецепт popover + @starting-style](https://pawelgrzybek.com/popover-element-entry-and-exit-animations-in-a-few-lines-of-css/), на самом деле сгодится для любых «модалок» и появляшек: `@starting-style` привносит в этот паттерн недостающую декларативность, чтобы не приходилось вручную манипулировать классами/эвент-хэндлерами
- да и вообще, почему бы просто не [добавить @starting-style ко всему](https://codepen.io/argyleink/pen/Povqdej)
  `* { transition: opacity .5s ease-in; @starting-style { opacity: 0 } }`?

**HTML**

- [статья по мотивам недавнего опроса State of HTML 2023](https://frontendmasters.com/blog/state-of-html-2023-results-2/): людям нравится `datalist` (вывод: время адопта среднего HTML-элемента — 5 лет), не нравятся формы (стилизация, инпуты, валидация, кроссбраузерность), люди помнят о существовании веб-компонентов (и немного используют), людям не хватает нативных элементов `datatable`, `tabs`, `switch/toggle`
- в 2023 людям не хватало switch, а [в 2024 он уже появился](https://www.smashingmagazine.com/2024/05/switching-it-up-html-latest-control/) (пока правда только в Safari 17.4, но тем не менее), `<input type="checkbox" switch />` можно стилизовать через `::thumb`, `::track`, но при этом он в отличие от чекбокса не может быть в indeterminate-состоянии и может быть required (в общем, ждём 5 лет и можно использовать)
