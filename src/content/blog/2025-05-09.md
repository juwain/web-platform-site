### Проекты

- [deepwiki](https://deepwiki.com) — проект, позволяющий для любого публичного репозитория сгенерировать с помощью AI документацию по исходному коду (создаются даже схемы и диаграммы): например, дока [mobx](https://deepwiki.com/mobxjs/mobx), [react](https://deepwiki.com/facebook/react), [linux](https://deepwiki.com/torvalds/linux); изучать новые либы стало ещё интереснее и проще
- [operator-lookup](https://www.joshwcomeau.com/operator-lookup/) — интерактивный туториал по операторам JS
- [pyodide](https://pyodide.org/en/stable/) — порт CPython на WebAssembly/[Emscripten](https://emscripten.org/), позволяющий запускать код на питоне в браузере/Node.js
- [bhvr](https://github.com/stevedylandev/bhvr) — а вот и попытка сделать просто работающий фуллстек-React-фреймворк, в который не нужно долго вкуривать, и он не залочен под конкретного вендора: под капотом **B**un, **H**ono, **V**ite, **R**eact
- [react-modal-sheet](https://github.com/Temzasse/react-modal-sheet) — неплохой UI-элемент выезжающей шторки под React
- [color.js](https://github.com/color-js/color.js) — либа для манипуляции с цветами от авторов спек (Lea Verou, Chris Lilley) и других уважаемых людей

### Статьи, мнения, туториалы

#### JS/TS

- [как организовать микрофронтенды с помощью Astro](https://medium.com/@sergio.a.soria/composing-micro-frontends-with-astros-server-islands-88a02728436c): каждый микрофронтенд организуется как отдельный Astro-сервер, с урлов всех микрофронтов получается HTML, а затем с помощью [Server Islands](https://docs.astro.build/en/guides/server-islands/) организуется отложенная загрузка динамики каждого фронта ([пример](https://github.com/sasoria/astro-microfrontends-ssr-distributed/tree/main))
- и снова рубрика «что хотел сказать автор» от Дэна Абрамова, [и снова про RSC](https://overreacted.io/rsc-for-astro-developers/): на примере того, как организован серверный и клиентский код в Astro, поясняется, что и как организовано в React; проблем сходу вижу две:
  - в Astro разделение на сервер/клиент сходу понятное и простое как палка
  - в React надо ещё уложить в голове, что компоненты по-умолчанию серверные, а могут быть клиентские, а на самом деле могут быть и такими, и такими
- пара человеческих статей о том, как по-разному устроена работа эвент лупа [в браузерном JS](https://www.deepintodev.com/blog/how-javascript-works-behind-the-scenes) и в [Node.js](https://www.deepintodev.com/blog/how-nodejs-works-behind-the-scenes)
- [как примерно устроен санитайзер HTML](https://waspdev.com/articles/2025-05-07/how-to-sanitize-html-without-libraries):
  - берём сырой HTML,
  - вставляем его в новый неактивный документ через [DOMParser](https://developer.mozilla.org/en-US/docs/Web/API/DOMParser),
  - затем пробегаемся рекурсивно по дочерним нодам, смотрим на соответствие их списку допустимых тегов, а также на _нессылки_ в урлах ссылок или картинок,
  - берём проверенные ноды, отбрасываем не прошедшие проверку
  - возвращаем отфильтрованные ноды

#### CSS

- [сборник юзкейсов](https://frontendmasters.com/blog/shape-a-new-powerful-drawing-syntax-in-css/) новой функции `shape()` (ждём только [FF в 141 версии](https://caniuse.com/mdn-css_types_basic-shape_shape)), и заодно [интро](https://webkit.org/blog/16794/the-css-shape-function/) в блоге webkit: самый сок этой функции — это человекочитаемые команды внутри и возможность использовать переменные, чтобы делать формы респонсивными
- хорошая фича [margin-trim](https://webkit.org/blog/16854/margin-trim/), которая подрезает лишние отступы детей, выезжающие за родителя, но поддерживается только в Safari, печаль
- [утилитарный класс](https://gomakethings.com/an-aspect-ratio-css-utility-class/) для установки `aspect-ratio` у картинок/видео, само соотношение сторон в переменной, чтобы задать извне:

```css
[class*="aspect-ratio"] {
  --ratio: auto;
  object-fit: cover;
  aspect-ratio: var(--ratio);
}
```

- подход к решению задачи показа/сокрытия боковых фейдов при скролле (в начале скролла фейд скрыт со стороны начала скролл-контейнера, в конце — со стороны конца контейнера): обычно это решается через JS, но [здесь приводится решение через скролл-анимации](https://css-tricks.com/modern-scroll-shadows-using-scroll-driven-animations/), когда в зависимости от кейфрейма, переменная обнуляется или задаётся значением:

```css
@keyframes scrollfade {
  0% {
    --left-fade: 0;
  }
  10%,
  100% {
    --left-fade: 3rem;
  }
  0%,
  90% {
    --right-fade: 3rem;
  }
  100% {
    --right-fade: 0;
  }
}
```

### Платформа

- вот уже [пошли первые фантазии](https://thenewstack.io/frontends-next-evolution-ai-powered-state-management/) на тему AI-мидлвар в стейт-менеджерах на фронте: типа вместо написания конкретной логики делегируем разрешение конфликтов, мемоизацию и префетч умной плослойке в виде AI-модели (уходим от кодоцентричной разработки к описанию поведения на естественном языке)
- часто [мы не пишем чистый код](https://cekrem.github.io/posts/psychology-of-clean-code/), потому что торопимся успеть до дедлайна, перегружены или не хотим трогать, чтобы не поломалось; но в целом, если устраивать «партизанский» рефакторинг, ревьюить самого себя до отправки на общественное ревью и представлять, что скажет «будущий я» по поводу этого кода, то код точно станет получше
