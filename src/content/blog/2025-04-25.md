### Статьи и демки

#### JS/TS
- в недавно вышедшем Chrome 135 [появилась экспериментальная поддержка fetchLater](https://chromestatus.com/feature/4654499737632768) — особого API для отправки beacon-запроса «в один конец»; замысел авторов — не отталкиваться больше от событий страницы (`unload`/`beforeUnload`, которые поддерживаются нестабильно), а перейти к регистрации разработчиками намерения отправить beacon-запрос, а браузер уже позаботится об остальном сам (хотя, в целом, целесообразность появления этого API не очень ясна, ведь недавно как раз везде поддержали `keepAlive` у обычного `fetch`)
- одна из причин, почему [с веб-компонентами всё сложно](https://plainvanillaweb.com/blog/articles/2025-04-21-attribute-property-duality/): атрибут у HTML-элемента и значение свойства у JS-объекта этого элемента — это разные штуки: когда меняется свойство, значение атрибута не меняется, а вот смена значения атрибута чаще всего отзеркаливается в смене значения свойства, и в веб-компонентах как кастомных элементах с этим нужно разбираться вручную
- [замена ветвистых условий](https://simonsafar.com/2025/if_statements/) в функциях на плоскую структуру, где сначала идут early-returns, после идёт непоредственно функциональность, а затем обработка ошибок выполнения, обычно улучшает читаемость, и хочется меньше скипать код при его изучении
- [разбор более сложных кейсов в TS](https://piccalil.li/blog/advanced-typescript-manipulation-features-for-the-real-world/): `as const` для уточнения типа, indexed access types, mapped types:
```ts
const sendEvent = <E extends EventName,>(
	event: E,
	payload: HomeEvents[E]
) => { /* ... */ };
```
 
```ts
type BooleanProperties<Type> = {
	[Key in keyof Type]: boolean;
};
```

- [история про новенький Cookie Store API](https://fotis.xyz/posts/the-new-cookie-store-api/), который то ли уже появился в оставшемся FF, то ли всё таки ещё нет, и о том, как уже сейчас можно встроить его в React-приложение с фолбеком
- использование `useEffect` для синхронизации нескольких состояний — [не очень хорошая идея](https://ondrejvelisek.github.io/avoid-state-synchronization-trap/)

#### CSS
- сердечко с [новомодной функцией `shape()`](https://css-tip.com/heart/)
- если вы не знаете, [как подступиться к формату oklch](https://adactio.com/journal/21853), имея на руках макеты с rgb-цветами, то можно начать с «каста» rgb-цвета в oklch, а дальше уже тюнить lch-результат:
```css
--page-colour: oklch(from #49498D calc(l / 4) c h);
```

### Платформа
- помимо привычных браузеров Chrome, Firefox, Safari и Edge, которые в современном мире распространяются дефолтными установками в ОС, на базе их опенсорсных движков строят [другие браузеры со своими фишками](https://frontendmasters.com/blog/newfangled-browser-alternatives/): где-то с переосмысленным UX (Arc, Horse, Zen, Wavebox, Surf, Shift), где-то с доп privacy (Orion, DuckDuckGo), где-то с доп функциями (Vivaldi), где-то с криптой (Brave), но отдельные смельчаки строят полностью свои движки (Ladybird, Flow, Servo)
- [ещё один пойнт](https://cekrem.github.io/posts/coding-as-craft-going-back-to-the-old-gym/) про осознанное применение AI в кодинге: делегируя AI-помощнику трудности решения проблем и нахождение решений, вы обкрадываете свой же опыт, благодаря которому нарабатывается экспертиза (преодоление «диких» условий без сторонней помощи закаляет), а также лишаете себя удовольствия от процесса (ведь кодинг это не только выдача готового кода, но и удовлетворение от становления лучшей версией себя в процессе его написания), а вот делегировать рутинные части, бойлерплейт и объяснение концепций — вполне ок для AI
