**#Пульс*веб*платформы 019.07.2024**

**Новости**

- [GitLab ищет, кому продаться](https://www.reuters.com/markets/deals/google-backed-software-developer-gitlab-explores-sale-sources-say-2024-07-17/): конкурировать с GitHub с копайлотом и нейросетям не вариант, пора выводить вложения; факт — пуллреквесты победили мерджреквесты
- [вышел Deno 1.45](https://deno.com/blog/v1.45): среди прочего поддержаны [npm workspaces](https://docs.npmjs.com/cli/v7/using-npm/workspaces) (ну и конечно представлен свой кошерный аналог), так что работа с монорепами всячески улучшается с разных мест индустрии
- разработчики [jQuery готовятся выпустить вторую бету](https://blog.jquery.com/2024/07/17/second-beta-of-jquery-4-0-0/), в которой переработали тесты: в нынешних реалиях нужны [смоук-тесты](https://github.com/jquery/jquery/pull/5429/files#diff-538a3cbad913567d5f6ff7b3f521e73b8784328305e312a58fa01d158fb0c6dc) для проверки, что проект собирается на webpack, rollup с виде cjs + esm, и чистом esm, а также импортируется в node
- [Astro и Netlify теперь партнёры](https://astro.build/blog/netlify-official-deployment-partner/), будут получать ежемесячно денежку за то, что мягко приземляют проекты в облако; считаю, это хорошо, Astro — крутой проект
- в хромиум-браузерах [заработал copy-paste SVG-элементов](https://blogs.windows.com/msedgedev/2024/07/11/seamless-svg-copy-paste-on-the-web/)

**Проекты**

- [tinybase](https://tinybase.org/) — (в догонку к прошлому выпуску фронтвестника) local-first хранилище данных с возможностью синхронизации с сервером, минимального размера 5-15кб в зависимости от опций, хранит данные в виде пар ключ-значение или таблицы
- [act](https://github.com/nektos/act) — запускальщик GitHub Actions локально, чтобы не гонять CI почём зря

**Статьи и демки**

**JS**

- [как на коленке собрать TanStack Query](https://www.bbss.dev/posts/react-learn-suspense/), а также `Suspence` c хуком `use()`: внутри кеши с `Map` и `Set`, промисы и использование их полей `status`, `reason` и `value`, а также контекст для распространения
- история о том, как от идеи хранения глобального состояния в React-контексте (чтобы задействовать Concurrent Mode) автор перешёл к идее отдельного стора, который по необходимости контекстом только распространяется, но не «живёт» в нём, и [так родился современный Zustand](https://blog.axlight.com/posts/how-zustand-was-born/), заботливо выращенное дитя, который, верю, ждёт хорошее будущее (уже скоро Zustand 5)
- для [лучшей производительности интерфейса](https://kurtextrem.de/posts/improve-inp), рекомендуют разбивать большие таски, загружающие тред, на более мелкие, откладывая из выполнение с помощью `setTimeout`, `requestAnimationFrame`, а также нового API `scheduler.yield`; также можно откладывать выполнение служебных задач (аналитика и тд) до срабатывания события страницы `visibilitychange`
- `['1', '5', '11'].map(parseInt)` вернёт `[1, NaN, 3]`, тк вместе `parseInt` вместе с числом заодно получит вторым параметром индекс элемента, который будет использоваться в качестве системы счисления; вывод: так себе идея прокидывать [в map напрямую «служебные» функции](https://dev.to/safdarali/why1511mapparseint-returns-1-nan-3-in-javascript-2mhm), внутри неявность, что плохо

**CSS**

- из [всех способов спрятать что-то на странице](https://gomakethings.com/the-many-ways-to-hide-things-in-the-dom/) самый удобный — атрибут `hidden`, если не хочется подключать CSS; `visibiliy: hidden` и `display: none` скрывают элементы также визуально и от скринридеров; и ещё парочка способов для сокрытия только от скринридеров и для видимости только им
- [радиальный градиент внутри маски](https://codepen.io/jh3y/pen/gOeXgXv) + карусель с картинками = эффект линзы, элегантный, чисто визуальный трюк с беспощадным кодом
- объявление кастомных свойств через директиву `@property` (которая недавно, наконец, стала поддерживаться во всех браузерах) мало того, что открывает [возможность анимировать переменную](https://web.dev/blog/at-property-baseline), так ещё и даёт [лучшую поддержку типизации в браузере](https://css-tip.com/typed-variables/) (да, в CSS тоже есть типы) + ещё есть, например, немного фриковая возможность [получить пиксельную ширину/высоту экрана](https://css-tip.com/screen-dimension/) `--w: tan(atan2(var(--_w),1px))`
- вендорные префиксы были плохим решением для внедрения экспериментальных фич, а вот с [фича-флагами вышло уже хорошо](https://chenhuijing.com/blog/how-to-try-experimental-css-features/): они доступны повсеместно во всех браузерах, чтобы попробовать экспериментальные фичи

**HTML**

- [инициатива](https://utilitybend.com/blog/an-update-on-invokers-invoker-commands-in-html) добавить в стандарт атрибуты commandfor и command, чтобы унифицировано связывать интерактивные элементы прямо в HTML: `<button commandfor="custom-video" command="play">` запустит `<video id="custom-video">` ([полный список предложений](https://open-ui.org/components/invokers.explainer/#defaults) включает `dialog`, `input`, `audio/video`, `select`, `details`…)

**Платформа**

- [кулстори](https://conduition.io/coding/ticketmaster/) о том, что технологические барьеры — не инновация, а бесячья тупость; о том, что offline-first — лучший подход, так как он имитирует реальность; о том, что веб — открытая среда, в которую несложно внедриться для реверс-инжиниринга (особенно когда клиент-сервер пересылают в открытую токены для генерации TOTP-строки, которая является, к примеру, билетом на концерт)
