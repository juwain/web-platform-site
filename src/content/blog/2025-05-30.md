### Новости

- [закрывается браузер Arc](https://browsercompany.substack.com/p/letter-to-arc-members-2025): создатели переключились на другой браузер (с AI) Dia, под капотом тоже Chromium и проприетарная платформа для создания браузеров, которую опенсорсить не собираются
- также [закрывается проект Glitch](https://blog.glitch.com/post/changes-are-coming-to-glitch/) (браузерный редактор веб-приложений + хостинг) после 10 лет работы
- [в ESLint провели ретроспективу](https://eslint.org/blog/2025/05/eslint-v9.0.0-retrospective/) выкатки flat-config-а с 2019 по 2024, выявили основные ошибки: слишком много ломающий изменений за раз, слишком много документации и отсутствие тулинга автомиграции, медленное впитывание новой версии экосистемой, отказ от двойных конфигов (надо было поддержать, востребовано авторами плагинов)
- господа из React Router [поддержали в RR React Server Components](https://remix.run/blog/rsc-preview), а также [решили воскресить Remix](https://remix.run/blog/wake-up-remix) (в лучших традициях рестлинга) да не просто так, а сделать его полностью независимым, даже от React (внутри будет форк Preact): теперь стало понятно, что этот проект — экспериментальная площадка, чтобы проверять крейзи-фичи и затем интегрировать их в «стабильный» RR
- [вышел Chrome 137](https://developer.chrome.com/release-notes/137):
  - поддержали функцию `if()` в CSS: `background-color: if(style(--color: white): black; else: white)`
  - внедрили свойства `reading-flow`, `reading-order` для указания порядка чтения элементов скринридерами в флекс-/грид-лейаутах
  - заработала функция `shape()` для указания направления в свойстве `offset-path`
  - для SVG-элементов теперь можно управлять трансформацией через атрибут `transform`
  - добавили значение `view-transition-name: match-element`, чтобы автогенерировать название вью-транзишна вместо хардкода или задания из JS
- [вышел Firefox 139.0](https://www.mozilla.org/en-US/firefox/139.0/releasenotes/):
  - из коробки работает Temporal proposal (прикол в том, что для движка SpiderMonkey [поддержку реализовал](https://spidermonkey.dev/blog/2025/04/11/shipping-temporal.html) один чел, опенсорсный контрибьютор, который не работает в Mozilla)
  - поддержали атрибут `hidden=until-found`
  - в экспериментальных фигах за флагом приехали: View Transition API, [`scheduler.yield()`](https://developer.mozilla.org/en-US/docs/Web/API/Scheduler/yield)
- [вышла новая версия Bun v1.2.14](https://bun.sh/blog/bun-v1.2.14):
  - позаимствовали фичу `catalogs:` из pnpm для создания алиасов для группы зависимостей
  - добавили флаг `bun init --react` для быстрого разворачивания шаблона React-проекта и дев-сервером на Bun
- [вышла новая версия JSPM 4.0](https://jspm.org/jspm-4.0-release) (import map package manager): внутри команды для безконфиговой сборки и сервинга проекта, поддержка нативного TS «со стриппингом», импортмапы средствами браузера — звучит знакомо? да, потому что контрибьютор JSPM и [ES Module Shims](https://github.com/guybedford/es-module-shims) — один и тот же человек, Guy Bedford
- [вышел Angular 20](https://blog.angular.dev/announcing-angular-v20-b5c9c06cf301):
  - стабилизированы реактивные фичи `effect`, `linkedSignal` и `toSignal`
  - добавлена экспериментальная поддержка фетчера `httpResource()`
  - причёсан SSR

### Статьи, мнения, туториалы

#### JS/TS

- реактовский `useSyncExternalStore` можно использовать не только для хранения каких-то бизнес-данных (типичный стор), но и [для «UI»-информации](https://www.epicreact.dev/use-sync-external-store-demystified-for-practical-react-development-w5ac0), например, данных о `window.matchMedia`: подписываемся на `'change'` состояния экрана, записываем в «стор», в React-компоненте юзаем этот стор
- [как избегать проблемы циклических импортов](https://amorgunov.com/posts/2025-05-20-how-solve-cross-imports/) между компонентами с помощью здравого смысла, а также инверсии зависимостей (меняем прямое использование на пропы) на примере архитектуры FSD
- в связи с начавшейся раскаткой Temporal в браузерах (FF — первый) [пора знакомиться](https://waspdev.com/articles/2025-05-24/temporal-api), если ещё не, тем более, есть [полифилл](https://www.npmjs.com/package/temporal-polyfill)
- [краткая история JS](https://deno.com/blog/history-of-javascript) с уклоном в Райана Даля, Node и Deno: в 2025 JS исполнилось 30 лет, JSDoc — 26 лет, JSON — 24, Safari — 22, MDN — 20, jQuery — 19, Chrome — 17, CommonJS, Node.js, CoffeeScript — 16, npm, AngularJS, Backbone — 15, Webpack, TypeScript — 13, React — 12, ES6 — 10, Next.js — 9, Bun — 2

#### CSS

- помимо именования view-transions с ними есть [ещё одна проблема](https://benfrain.com/first-adventures-in-view-transitions/): так как транзишн применяется к `document`, одновременно может выполняться только один транзишн, поэтому невозможно запустить конкурентные транзишны + есть проблема с `z-index`, когда транзишнящийся элемент перекрывает всё вокруг; решение — [будущий Scoped View Transitions](https://github.com/WICG/view-transitions/blob/main/scoped-transitions.md), то есть возможность запуска транзишна на конкретном элементе `element.startViewTransition()`
- [эффект «фонарика в темноте»](https://frontendmasters.com/blog/css-spotlight-effect/): что прикольно, так это, что можно задавать любую форму светлому участку благодаря `filter` или `mix-blend-mode`, например, «решётка» из градиентов сильно смазанная и переконтращенная `filter: blur(1em) contrast(100)` становится движущейся «кляксой»
