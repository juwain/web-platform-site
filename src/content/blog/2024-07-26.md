**Новости**

- [в Google Docs завезли](https://workspaceupdates.googleblog.com/2024/07/import-and-export-markdown-in-google-docs.html) полноценную поддержку markdown (импорт/экспорт/копипейст)
- [вышел Chrome 127](https://developer.chrome.com/blog/new-in-chrome-127): поддержано свойство `font-size-adjust` (костылим везде, [теперь поддерживают все браузеры](https://web.dev/blog/font-size-adjust)!), alt-текст в content, Document picture-in-picture теперь общается с основным окном, на контейнеры со скроллом теперь можно сфокусироваться с клавиатуры
- в React завозят server-only плюшки: [отдельный пакет renderToMarkup](https://github.com/facebook/react/pull/30105) предназначен для асинхронного рендера JSX в статику, которая не будет дальше никак обновляться (например, для генерации писем и другой статики); использование state, effect будет сыпать ошибки (впрочем, всё же сделали [отдельный пакет для клиент-сайда](https://github.com/facebook/react/pull/30121), пометив сразу «для поддержки легаси-сред»)
- вышли результаты [Stackoverflow Developer Survey 2024](https://survey.stackoverflow.co/2024/): веб всё ещё на коне, хотя зарплаты, как и почти везде (за исключением Erlang) уменьшились (при этом фронтендеры — почти самые низкооплачиваемые IT-специалисты), в БД лидирует PostgreSQL, jQuery популярнее Next.js, Markdown File — третий по популярности инструмент для асинхронной коммуникации
- опубликованы [результаты опроса State of React 2023](https://2023.stateofreact.com/): RCS заходит туго, боли вызывает forwardRef, Redux и тот же RCS, популярны Tanstack Query, Zustand, Astro, Radix, shadcn/ui, Next.js; доминирует по прежнему SPA

**Проекты**

- [swapy](https://swapy.tahazsh.com/) — создание в приложении виджетного лейаута с драг-н-свапом
- [designgui](https://www.designgui.io/) — расширение Chrome для просмотра и управления CSS-переменными с цветами темы (как минимум, чтобы поудобнее искать и копировать цвета)
- [svg.wtf](https://svg.wtf/) — инструмент для изучения и экспериментирования с SVG

**Статьи и демки**

**JS**

- [Svelte дозрел](https://frontendmasters.com/blog/introducing-svelte-5/) до того возраста, когда уже поднакопилось достаточно массы, чтобы нужен был двусторонний interop между двумя мажорными версиями (читать как «достаточно наворотили»), в то же время изменения к лучшему в понятности API в новой версии налицо (читать как «исправляют навороченное, но надо поддерживать старый синтаксис»); в целом, впечатления о нём странные: что-то похожее на vue c mobx, при этом с костыльками для работы с TS, типа `<script lang="ts" generics="T extends { name: string }">`
- когда читаешь про всякие WASM SQLite, Origin Private File System (OPFS) и их использование через SharedWorker и WebWorker, обычно это всё кажется какой-то далёкой от реальности фантастикой, только если ты не работаешь в Notion и тебе надо [ускорить навигацию между страницами](https://www.notion.so/blog/how-we-sped-up-notion-in-the-browser-with-wasm-sqlite), сохраняя бд в файловой системе устройства и шаря её на все открытые табы
- [отличия «объектов» в TS](https://webdeveloper.beehiiv.com/p/differences-object-object-typescript): Object — это любые объекты типа `string`, `boolean`, `number`, `bigint`, `symbol` (у них есть свойства `cunstructor`, `toString`…), `{}` — это пустой объект без полей, а `object` — не непримитивные типы, то есть `{}`, `[]` и `() => {}` (`object` также имеет аналог `Record<string, any>`)
- [напоминание](https://swizec.com/blog/dry-the-common-source-of-bad-abstractions/): DRY как самоцель может подтолкнуть к созданию ненужных абстракций, которые часто оказываются ещё и негибким, а гибкость — наверное одно из важных качеств абстракций
- [хук useId](https://reacttraining.com/blog/use-useid-instead-of-hand-making-ids) можно использовать для генерации id DOM-элементов в рантайме

**CSS**

- в гайдах от Ahmad Shadeed обычно интересны практические юзкейсы использование фич; вот и в [этом гайде по grid-area](https://ishadeed.com/article/css-grid-area/) приводятся жизненные примеры использования: перестановка блоков местами в лейауте на разных разрешениях, «вылезающие» из прямоугольной сетки блоки, накладывающиеся области грида, has()-комбо `.card:has(figcaption) { grid-template-areas: 'update layout'}`
- [схема](https://css-tip.com/scroll-progress/), которая с появлением scroll-based-анимаций, стала ещё актуальнее: кастомное свойство — стейт, анимация — способ изменения стейта, счётчик — способ отображения стейта
- [напоминание](https://frontendmasters.com/blog/single-directionally-allowed-overflow/): контейнер с `overflow: hidden` всё равно можно проскроллить выделением текста или программно, а с overflow: clip возможность скролла будет гарантировано полностью выпилена
