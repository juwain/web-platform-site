### Новости

- команда TS выкатила [превью нативного TS на Go](https://devblogs.microsoft.com/typescript/announcing-typescript-native-previews/), можно попробовать в терминале `npm install -D @typescript/native-preview` или в [VS Code в виде расширения](https://marketplace.visualstudio.com/items?itemName=TypeScriptTeam.native-preview): оно живое (!) и по ощущениям LSP в VS Code шевелится побыстрее

- и тему VS Code продолжает новость, что в редакторах HTML и CSS в подсказках будет выдаваться инфа о Baseline-поддержке свойств, тегов и атрибутов (а также не только в VS Code, но и в других редакторах, которые используют [опенсорсные наработки Code - OSS](https://github.com/microsoft/vscode#readme))
- [в PHP 8.5 появится pipe-оператор](https://www.amitmerchant.com/the-pipe-operator-php-85/) (ноябрь 2025), а вот [подзаброшенный пропоузал в JS](https://github.com/tc39/proposal-pipeline-operator) уже вряд ли вскоре оживёт
- [Stackoverflow стремительно теряет активных пользователей](https://devclass.com/2025/05/13/stack-overflow-seeks-rebrand-as-traffic-continues-to-plummet-which-is-bad-news-for-developers/) (и не потому что на все вопросы уже ответили, а потому что узнавать ответы стали в других местах), поэтому компания думает о ребрендинге и перефокусировке с только Q&A ещё на сообщество и построение карьеры; что ж, такие времена, или адаптируйся под AI, или умри
- [Mozilla просит суд](https://blog.mozilla.org/en/mozilla/internet-policy/amicus_brief/) не перекрывать «денежный» кислород от Google в антимонопольном деле; такой вот парадокс: чтобы поддержать хоть какую-то конкуренцию, нужно продолжать делать «нерыночные» вливания за поиск Google по умолчанию в FF
- ESLint в добавок к JSON, md и CSS, теперь умеет линтить HTML [с помощью плагина html-eslint](https://eslint.org/blog/2025/05/eslint-html-plugin/); как и в прошлых случаях из коробки сразу идут многие базовые правила, которые не надо дополнительно настраивать
- [вышли ES Module Shims 2.5](https://guybedford.com/hot-reloading-es-module-shims-2.5) (это те, которые уже в версии 2.0 давали возможность запускать type-erasable-код на TS прямиком в браузере, без компиляции):
  - позволяют использовать нативные импорты CSS- и JSON-модулей: `import style from './style.css' with { type: 'css' };`
  - нативный TS теперь просто работает, без доп настроек
  - появилась возможность замутить «нативный» hot reload через `import.meta.hot`
- в Deno начали тыкать палочкой, типа их фреймворк Fresh что-то уже не особо-то и fresh (протух-с), поэтому [вышел пост](https://deno.com/blog/an-update-on-fresh), что Fresh 2.0 уже на подходе, под капотом будут Express/Hono-like API, позволяющие без лишнего бойлерплейта писать сервер, организовать мидлвари и другие плюшки
- в семействе Rs- [вышла новая либа для создателей либ — Rslib](https://lib.rsbuild.dev/blog/introducing-rslib): внутри уже предустановленные конфиги для беспроблемной сборки (в том числе стилей) и публикации либ в разных форматах (ESM, CJS, UMD, Module Federation)
- а в семействе TanStack [вышла клиентская БД/стор — db](https://github.com/TanStack/db), которая держит данные на клиенте, позволяет делать выборки (query) из этой БД и синкается с любым бэком

### Статьи, мнения, туториалы

#### JS/TS

- для работы с буфером обмена можно использовать методы [`navigator.clipboard.writeText()`](https://developer.mozilla.org/docs/Web/API/Clipboard/writeText) для обычного текста или [`navigator.clipboard.write()`](https://developer.mozilla.org/docs/Web/API/Clipboard/write) для всех остальных форматов типа картинок, текстового контента и HTML; а проверить браузер на умение вставлять определённый формат (без выброса исключения) можно с помощью [`ClipboardItem.supports()`](https://developer.mozilla.org/docs/Web/API/ClipboardItem/supports_static)
- а собственно [Async Clipboard API](https://web.dev/articles/async-clipboard) поддерживает помимо стандартных `text/plain`, `text/html` и `image/png` ещё и кастомные форматы: для этого при создании `new ClipboardItem` нужно указать в названии типа префикс `web `:

```js
const clipboardItem = new ClipboardItem({
  [`web ${jpegBlob.type}`]: jpegBlob,
  [`web ${gifBlob.type}`]: gifBlob,
});
```

- [юзкейсы генераторов в JS](https://macarthur.me/posts/generators/) обычно довольно специфические, но если свести к сути, то они подходят для любой условно «бесконечной» серии действий: перебора элементов, генерации наборов, обработки асинхронных сообщений; неочевидный пример:

```js
function* getElements(tagName = "div") {
  while (true) yield document.createElement(tagName);
}
```

#### CSS

- alt-текст, который показывается, если изображение не загрузилось, [тоже можно стилизовать](https://piccalil.li/blog/you-can-style-alt-text-like-any-other-text/): для этого к `img {}` просто применяются текстовые стили, можно дополнительно проставить в JS data-атрибут для картинок на колбеке `onerror`
- для нативных `<input type="date">` и `<input type="time">` в _некоторых хромиум-браузерах_ показываются иконки часов и календаря, для [их стилизации](https://cassidoo.co/post/input-type-date/) подходит псевдокласс `::-webkit-calendar-picker-indicator`
- если решение работает, это не значит, что оно оптимальное; так ответ на StackOverflow может быть устаревшим, хоть и популярным, и чат-бот также может взять популярное устаревшее решение из поискового топа; вот к примеру [для анимированного градиента текста](https://frontendmasters.com/blog/chatgpt-and-old-and-broken-code/) достаточно таких стилей:

```css
.gradient-text {
  background: linear-gradient(90deg, #00f, #0ff, #00f) -100%/ 200%;
  color: transparent;
  animation: shimmer 2s linear infinite;
  -webkit-background-clip: text;
  background-clip: text;
}

@keyframes shimmer {
  to {
    background-position: 100%;
  }
}
```

- новую функцию `shape()` можно использовать не только в связке с `clip-path()` для рисования фигур, но и с `offset-path` [для задания формы движения](https://frontendmasters.com/blog/move-modal-in-on-a-shape/): вместо `offset-path: path("")` более читаемый `offset-path: shape()`
- [некоторые CSS-сниппеты](https://adactio.com/journal/21896) разной степени полезности: точно подтвержу, что если делаете какой-то общий компонент со стилизацией, которая может внезапно стать «глобальной» (стилизация по тегам `p`, `body`…), то её точно полезно завернуть в `@layer {}`, чтобы у потребителя не возникло необходимости переопределять ваши «глобальные» стили, так как они будут применяться раньше по каскаду
- [неинтуитивные решения для лейаута](https://nerdy.dev/3-unintuitive-layout-solutions), например, по умолчанию минимальная ширина grid- и flex-детей — `auto`, что не всегда удобно, и её можно сбросить:

```css
.wat {
  min-height: 0;
  min-width: 0;
}
```

#### HTML

- тег `<datalist>` можно использовать [не только для «выпадашки» текстовых значений](https://html-css-tip-of-the-week.netlify.app/tip/datalist/), таргет-элементом листа может быть, например, `<input type="color">`, а элементами листа, соответственно, цвета:

```html
<datalist>
  <option value="#ff0000">Red</option>
  <option value="#00ff00">Green</option>
</datalist>
```
