### Новости
- [вышел TS 5.8](https://devblogs.microsoft.com/typescript/announcing-typescript-5-8/), с момента RC изменений нет: `require(esm)` ( + флаг обратной совместимости `--module node18`) и флаг `--erasableSyntaxOnly` на борту
- вышел [Safari Technology Preview 214](https://webkit.org/blog/16512/release-notes-for-safari-technology-preview-214/): внутри долгожданная поддержка `data:` URL в favicon, то есть теперь можно будет во всех браузерах использовать SVG- и эмодзи-фавиконки:
```html
<link rel="icon" type="image/svg+xml" href="data:image/svg+xml,[YOUR ENCODED SVG HERE]" />
```
- вовсю идёт работа по [внедрению rolldown в Vite](https://github.com/vitejs/rolldown-vite)
- вышла стабильная версия [TanStack Form v1](https://tanstack.com/blog/announcing-tanstack-form-v1) для основных фреймворков:
	- TS валидирует связку модели данных и значения в компоненте, причём не нужно передавать дженерики, а всё инферится з коробки
	- встроеная поддержка валидационных либ (Zod, ArkType…)
	- есть возможность задать асинхронную функцию для валидации
- выпустили черновик спеки [CSS Form Control Styling Level 1](https://drafts.csswg.org/css-forms/) про стилизацию элементов форм: `appearance: base` для базовой стилизации элементов, псевдоэлементы `::picker`, `::thumb`, `::track`, `::fill` и другие для стилизации UI-элементов
- вышел [мартовский апдейт React Aria/Spectrum](https://react-spectrum.adobe.com/releases/2025-03-05.html) с компонентами [Toast](https://react-spectrum.adobe.com/react-aria/Toast.html), [Tree](https://react-spectrum.adobe.com/react-aria/Tree.html) и [Virtualizer](https://react-spectrum.adobe.com/react-aria/Virtualizer.html), также допилен хук [usePress](https://react-spectrum.adobe.com/react-aria/usePress.html) и компонент [Autocomplete](https://react-spectrum.adobe.com/react-aria/Autocomplete.html)
- вышел [Chrome 134](https://developer.chrome.com/release-notes/134):
	- добавили возможность добавлять внутрь `<select>` дополнительные элементы помимо `<option>`, `<optgroup>` и `<hr>`
	- допилили вариации закрытия `<dialog>`
	- дополнили обратносовместимо `console.timeStamp()`
- в [Chrome 135](https://groups.google.com/a/chromium.org/g/blink-dev/c/stxSgTgMHog?pli=1) будет внедрён [пропоузал Observable API](https://github.com/WICG/observable) (позиции WebKit и Mozilla по поводу этого API пока неясны):
```js
container
	.when('click')
	.filter((e) => e.target.closest('a'))
	.subscribe({
		next: (e) => {
			// …
		},
	});
```

### Статьи и демки

#### JS
- `parseInt('1️⃣') === 1 // true`, потому что внутри `parseInt` [находит в unicode-символе](https://www.aleksandrhovhannisyan.com/blog/parseint-keycap-emoji/) сначала `1`, а последующий нечисловой «вариационный» символ отбрасывается
#### CSS
- `<input type="range">` [превращается в «звёздный рейтинг»](https://css-tricks.com/a-css-only-star-rating-component-and-more-part-1/), чем удобно: использует нативное управления с клавиатуры, а также инпут имеет нативные атрибуты `min`, `max`, `step`, значения которых пробрасываются в CSS через новоиспечённый `attr`
- только привыкли к функциям-дженерикам в TS, а как [вам функции в CSS](https://css-tricks.com/functions-in-css/) (пока что почти что фэнтези-синтаксис)?
```css
@function --dashed-border(--color: red) {
  result: 2px dashed var(--color);
}

.argument {
  border: --dashed-border(blue);
}

.default {
  border: --dashed-border();
}
```
- вырезаем [прямоугольную прозрачную рамку](https://css-tip.com/inner-border/) внутри картинки с помощью `mask` и конического градиента
- кастомное свойство может быть [«выключено» по умолчанию](https://css-tricks.com/the-css-custom-property-toggle-trick/), правда это хак и синтаксис выглядит сломанным:
```css
button {
  --has-border: ; /* off by default */
  border: 1px solid var(--is-raised, rgb(0 0 0 / 0.1));
}

.bordered {
  --has-border: initial; /* turn on */
}
```
- [эффект калейдоскопа](https://nerdy.dev/css-kaleidoscopes) с помощью анимации кастомного свойства (`@property`) внутри градиента
