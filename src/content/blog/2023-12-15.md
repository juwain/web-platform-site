**#Пульс*веб*платформы 15.12.2023**

**Новости**

- вышел [Safari 17.2,](https://webkit.org/blog/14787/webkit-features-in-safari-17-2/) много мелких и классных обновлений в уходящий год, чтобы подбить статистику Interop 2023: аккордеон на `<details> + <summary>`, onetimecode для инпутов, «расслабленный» CSS-nesting, обновления [CSS Motion Path,](https://drafts.fxtf.org/motion-1/) функция `linear()` для создания «нелинейных» анимаций, математические функции `rem()`, `mod()`, `round()` в CSS, `display: list-item`, `mask-border` без префиксов, [Custom Highlights API,](https://developer.mozilla.org/en-US/docs/Web/API/CSS_Custom_Highlight_API) `<link rel="preload">` с атрибутами `imagesrcset` и `imagesizes` для подгрузки адаптивных изображений, `import` [attributes](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Statements/import) в JS, `fetchpriority` для загрузки ресурсов, улучшение нативной валидации форм и ещё много других плюшек
- [анонсирован](https://million.dev/blog/million-3) выход Million 3.0 — альтернативного VDOM для Реакта [(подробно о его работе):](https://million.dev/blog/virtual-dom) вкратце, он берёт для реконсиляции только динамически заполняемые DOM-ноды, а на статические не обращает внимания, а затем сравнивает не сами DOM-ноды, а их состояние (что существенно быстрее); годится для случаев, когда в приложении не слишком много динамических кусков, и часть приложения в ходе работы остаётся неизменной (значит её не нужно учитывать в пересчёте)
- вышел [SVGO v3.1.0,](https://github.com/svg/svgo/releases/tag/v3.1.0) улучшено всё, без маркетиноговых уловок сообщаю, что это просто классный проект, cli-тулза для чистки SVG от мусора
- Storybook с [8.0 версии](https://storybook.js.org/blog/storybook-react-server-components/) будет поддерживать React Server Components: минус — Next.js съест и Storybook, плюс — Storybook не будет медленно рендериться на клиенте

**Проекты**

- [Primer React](https://primer.style/react/) — библиотека Реакт-компонентов Гитхаба: круто, что такие проекты выкладываются в opensource, поэтому можно почитать исходники: из интересного, у них используются `styled-components` и чтобы компоненты писать вот так красиво `<Header.Item>` делается `Object.assign(Header, {Link: HeaderLink, Item: HeaderItem})` [(реф)](https://github.com/primer/react/blob/8a8973cd23e6810a15da5f0113395369d8bc3cd0/src/Header/Header.tsx#L73)
- [eruda](https://github.com/liriliri/eruda) — эмулятор браузерной консоли и девтулзов для мобильных браузеров
- [Fuse.js](https://stellate.co/blog/announcing-fuse-js) — фреймворк для создания типобезопасного слоя данных между бэком и фронтом, работает поверх Ноды и gql
- простой и честный респонсив компонент пагинации для Реакта — [React Responsive Pagination](https://github.com/jonelantha/react-responsive-pagination)
- [GQL (Git Query Language)](https://amrdeveloper.github.io/GQL/) — а как вам такое: можно делать выборки по истории гита с помощью SQL!
- [enhance](https://enhance.dev/) — HTML-first full-stack веб-фреймворк на веб-компонентах, выглядит очень симпатично, так и тянутся руки глотнуть свежего воздуха
